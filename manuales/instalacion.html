Características y componentes del Gestor de Base de datos

Gestor de base de datos Oracle

Oracle Database es un sistema de gestión de bases de datos relacionales de objetos desarrollado y comercializado por Oracle Corporation. Oracle Database se conoce comúnmente como Oracle RDBMS o simplemente Oracle.

Instancia de la base de datos

Una base de datos Oracle consta de una base de datos y al menos una instancia.

Una instancia, o instancia de base de datos, es la combinación de memoria y procesos que forman parte de una instalación en ejecución y una base de datos es un conjunto de archivos que almacenan datos.

A veces, una instancia de base de datos se conoce como una base de datos en ejecución completa. Sin embargo, es importante entender las distinciones entre los dos.

Primero, puede iniciar una instancia de base de datos sin tener que acceder a ningún archivo de base de datos. Así es como se crea una base de datos, comenzando primero una instancia y creando la base de datos desde dentro de la instancia.

En segundo lugar, una instancia solo puede acceder a una base de datos a la vez. Cuando inicia una instancia, el siguiente paso es montar esa instancia en una base de datos. Y una instancia puede montar solo una base de datos en un solo punto en el tiempo.

Tercero, varias instancias de bases de datos pueden acceder a la misma base de datos. En un entorno de clústeres, muchas instancias en varios servidores acceden a una base de datos central para permitir una alta disponibilidad y escalabilidad.

Finalmente, una base de datos puede existir sin una instancia. Sin embargo, sería inutilizable porque es solo un conjunto de archivos.

Estructuras de almacenamiento físico

Las estructuras de almacenamiento físico son simplemente archivos que almacenan datos. Cuando ejecuta una instrucción CREATE DATABASE para crear una nueva base de datos, Oracle crea los siguientes archivos:

Archivos de datos keyboard_arrow_down

Los archivos de datos contienen datos reales, por ejemplo, pedidos de ventas y datos de clientes. Los datos de las estructuras de bases de datos lógicas, como tablas e índices, se almacenan físicamente en los archivos de datos.
Más información...

Archivos de control keyboard_arrow_down

Cada base de datos tiene un archivo de control que contiene metadatos. Los metadatos describen la estructura física de la base de datos, incluido el nombre de la base de datos y las ubicaciones de los archivos de datos.
Más información...

Archivos de Log keyboard_arrow_down

Cada base de datos tiene un registro de Log que consta de dos o más archivos de registro de Log. Un registro de Log se compone de entradas de rehacer que registran todos los cambios realizados en los datos.

El repositorio de diagnóstico automático (ADR) es un repositorio jerárquico basado en archivos para manejar información de diagnóstico.

Los administradores de seguridad usan Oracle Wallet Manager para administrar credenciales de seguridad de clave pública en clientes y servidores Oracle.

Block changing tracking mejora el rendimiento de las copias de seguridad incrementales al registrar bloques modificados en el archivo de seguimiento de cambio de bloque. Durante una copia de seguridad incremental, en lugar de escanear todos los bloques de datos para identificar qué bloques han cambiado, RMAN usa este archivo para identificar los bloques modificados que necesitan copia de seguridad.

Estructuras lógicas de almacenamiento

Oracle Database utiliza una estructura de almacenamiento lógica para un control detallado del uso del espacio en disco. Las siguientes son estructuras de almacenamiento lógico en una base de datos Oracle:

  • Bloques de datos: un bloque de datos corresponde a una cantidad de bytes en el disco. Oracle almacena datos en bloques de datos. Los bloques de datos también se conocen como bloques lógicos, bloques de Oracle o páginas.
  • Extensiones: Una extensión es un número específico de bloques de datos lógicamente contiguos utilizados para almacenar el tipo particular de información.
  • Segmentos: un segmento es un conjunto de extensiones asignadas para almacenar objetos de base de datos, por ejemplo, una tabla o un índice.
  • Tablespaces: una base de datos se divide en unidades de almacenamiento lógicas llamadas tablespaces. Un tablespace es un contenedor lógico para un segmento. Cada espacio de tabla consta de al menos un archivo de datos.

Iniciar la instalación de Oracle 18C

Para iniciar el proceso de instalación ingresamos a la carpeta donde se encuentran los archivos para la descompresión y a continuación buscamos el arcivo de aplicación con nombre "setup".

Iniciamos el archivo con permisos de administrador.

El instalador iniciara el proceso de configuración de la nueva instancia de Oracle a instalar.

EL primer paso que se nos muestra es la elección de la clase del sistema que se desea instalar.

Aquí se nos presentan dos opciones de clase:

  • Clase de escritorio: elegimos esta opción si estamos instalando en un sistema de clase portátil o de escritorio. Esta opción incluye una base de datos de inicio y permite una configuración mínima.
  • Clase de servidor: elegimos esta opción si estamos instalando en un sistema de clase de servidor, que Oracle define como un sistema utilizado en un centro de datos de producción. Esta opción nos permite opciones de configuraciones más avanzadas

Seleccionamos la primera opción, Clase de escritorio, porque instalaremos Oracle Database 18c en un equipo de escritorio.

El siguiente paso nos muesta la configuración para el usuario de inicio de Oracle

El servidor Oracle crea cuatro grupos de usuarios de Windows:

  • Cuenta virtual de usuario: este tipo de cuenta es la instalación de instancia única de Oracle Home User para Oracle Database. La cuenta nos permite instalar Oracle Database, y crear y administrar servicios de bases de datos sin contraseñas.
  • Usar usuario de Windows existente: este tipo de cuenta puede ser un usuario local de Windows, un usuario de dominio de Windows o una cuenta de servicios administrados de Windows (MSA). Debemos proporcionar un nombre de usuario y contraseña para un usuario local o de dominio de Windows. Para la cuenta MSA, que es una cuenta de dominio administrado, solo necesitamos proporcionar el nombre de usuario. Para una instalación de Oracle RAC Database o Oracle Grid Infrastructure, solo podemos usar una cuenta de usuario de dominio de Windows.
  • Crear nuevo usuario de Windows: aquí debemos proporcionar el nombre de usuario y la contraseña para el usuario local de Windows que queremos que Oracle Universal Installer cree, y luego confirmar la contraseña. Al nuevo usuario que se crea se le niegan privilegios de inicio de sesión interactivo en la computadora con Windows. Sin embargo, un administrador de Windows puede administrar esta cuenta como otras cuentas de Windows.
  • Utilice la cuenta integrada de Windows: en este caso, no se requiere nombre de usuario ni contraseña. Oracle crea servicios de Windows utilizando cuentas integradas de Windows (LocalSystem o LocalService).

En ete caso seleccionamos la opción de crear un nuevo usuario con nombre Oracle

Lo siguiente es la configuración de la instalación de oracle.

El instalador nos pide una serie de instrucciones para configurar la base de datos a crear.

Oracle Base keyboard_arrow_down

El directorio de Oracle Base es un directorio de nivel superior para instalaciones de software de Oracle. Podemos establecer cualquier ubicación que queramos como nuestra Base Oracle.

Software Location keyboard_arrow_down

La ubicación del software es la ruta de inicio de Oracle donde se colocan los archivos binarios de Oracle Database para esta instalación. Antes de 18c se nos permitía establecer esta ubicación de Oracle Home, pero ahora, como podemos ver, la ubicación ya está configurada y no podemos modificarla.

El instalador toma la ubicación de los archivos de instalación y la asigna como la ubicación de Oracle Home. Por eso extrajimos nuestros archivos de instalación en un directorio separado.

Database File Location keyboard_arrow_down

La ubicación del archivo de la base de datos es la ubicación donde se almacenan los archivos de la base de datos de Oracle, como los archivos de datos, los archivos de registro de rehacer, los archivos de control, los archivos de parámetros del servidor y el archivo de contraseña. Estos son los archivos más importantes de Oracle Database. La ubicación predeterminada del archivo de datos es $ ORACLE_BASE / oradata para un solo nodo Oracle Real Application Clusters (Oracle RAC), instancia única e instalaciones de clase de escritorio. Para las instalaciones de Oracle RAC de múltiples nodos, la ubicación predeterminada es debajo de cualquier punto de montaje compartido descubierto en el conjunto de nodos seleccionado.

Database Edition keyboard_arrow_down

Edición de la base de datos: hay dos opciones aquí. Edición Enterprise y Standard Edition 2

Character set keyboard_arrow_down

A partir de Oracle Database 12c Release 2 (12.2), el conjunto de caracteres de la base de datos predeterminado de una base de datos creada a partir del Procesamiento de propósito general / Transacción o la plantilla de Almacenamiento de datos es Unicode AL32UTF8.

Unicode es el conjunto de caracteres universal que admite la mayoría de los idiomas que se hablan actualmente en el mundo. También es compatible con muchos scripts históricos (alfabetos). Unicode es la codificación nativa de muchas tecnologías, incluidas Java, XML, XHTML, ECMAScript y LDAP. Unicode es ideal para bases de datos que soportan Internet y la economía global.

Global Database Name keyboard_arrow_down

Global Database Name es el nombre que identifica la base de datos. Debe ser único para distinguir la base de datos de todas las demás bases de datos en una red.

El nombre de la base de datos global será el nombre de nuestra base de datos y servirá como nuestro SID de la base de datos. De forma predeterminada, está configurado como ORCL.

Global Database Name se compone de un nombre único de base de datos (db_unique_name), un delimitador (un punto) y un nombre de dominio de base de datos (db_domain). El nombre tiene el siguiente formato: db_unique_name.db_domain.

  • db_unique_name es el nombre de la base de datos. Puede contener un máximo de 30 caracteres, siempre que los primeros ocho caracteres sean únicos. Los caracteres pueden ser alfanuméricos (a-z, 0-9), guión bajo (_), dólar ($) y libra (#).
  • db_domain es el entorno informático utilizado para la base de datos. No debe contener más de 128 caracteres; alfanumérico (a-z, 0-9), guión bajo (_), libra (#) y puntos (.).
vpn_key Mínimos

Requisitos mínimos de contraseña

  • La contraseña no puede exceder los 30 caracteres.
  • No hay campos de contraseña vacíos.
  • El nombre de usuario no puede ser la misma cadena de caracteres que una contraseña.
  • La contraseña de la cuenta SYS no puede ser change_on_install. (sin distinción entre mayúsculas y minúsculas)
vpn_key Recomendados

Recomendaciones de contraseña

  • Contiene al menos una letra minúscula.
  • Contiene al menos una letra mayúscula.
  • Contiene al menos un dígito.
  • Tiene al menos 8 caracteres de longitud.
  • Utiliza el conjunto de caracteres de la base de datos, que puede incluir los caracteres de subrayado (_), dólar ($) y signo de número (#).
  • No debería ser una palabra real.

Desinstalar la base de datos oracle

Proceso de desinstalación de la base de datos Oracle 18c.

Para la correcta eliminación de todos los archivos de Oracle 18C sigua los siguientes pasos:

1. Asegurarse que se poseen permisos de administrador para proceder con la desintalación

2. Eliminar las Variables de entorno

  • Ingrese a las "Propiedades del sistema" > "Propiedades avanzadas del sistema"
  • Seleccionar "Variables de entorno"
  • Eliminamos la que redirija a la ruta creada de instalación D:\db_home/bin

3. Eliminar el registro de Oracle 18c

  • Click Windows + R
  • Ingresar REGEDIT
  • Buscamos el directorio con nombre HKEY_LOCAL_MACHINE > SYSTEM > CurrentControlSet > Services
  • Buscamos las carpetas que contengan Oracle
  • Eliminamos una por una las 6 carpetas
  • Buscamos el directorio HKEY_LOCAL_MACHINE > SOFTWARE
  • Eliminamos la carperta ORACLE

4. Reiniciar el sistema

5. Eliminar el usuario de Oracle y El grupo Oracle

  • Click Windows + R
  • Ingresar lusrmgr.msc
  • Eliminamos User > Nombre del usuario de oracle
  • Eliminamos Groups > Todos los que contengan ORA (9)

6. Eliminar la carpeta donde estan los archivos de oracle "db_home"

7. Eliminar del disco principal la carpera Oracle c:\Program Files/Oracle

8. Reiniciar
